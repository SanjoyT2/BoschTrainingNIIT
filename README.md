# Bosch Training NIIT

This repository contains:

- `defects_data.csv` — defect dataset used for EDA and modeling
- `defects_eda.ipynb` — exploratory data analysis notebook
- `fibonacci.ipynb` — sample Fibonacci notebook
- `app.py` — FastAPI app exposing prediction endpoints
- `requirements.txt` — Python dependencies
- `severity_model.pkl` — serialized severity prediction model
- `cost_model.pkl` — serialized repair cost prediction model

## Setup

Install dependencies with:

```bash
pip install -r requirements.txt
```

## Run EDA and train models

Open `defects_eda.ipynb` in Jupyter and run the notebook cells. The notebook includes:

- data loading and cleaning
- EDA plots and summaries
- model training for severity prediction
- model training for repair cost prediction
- saving the trained models to `severity_model.pkl` and `cost_model.pkl`

## Run FastAPI service

Start the API server with:

```bash
python -m uvicorn app:app --reload --port 8000
```

### Endpoints

- `GET /health` — check model availability
- `POST /predict/severity` — predict severity label
- `POST /predict/cost` — predict repair cost

### Example request

```bash
curl -X POST "http://127.0.0.1:8000/predict/severity" \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": 10,
    "defect_type": "Functional",
    "defect_location": "Component",
    "inspection_method": "Manual Testing",
    "month": 5,
    "weekday": "Friday",
    "is_weekend": 0,
    "product_defect_count": 4
}'
```

## Notes

- Ensure `severity_model.pkl` and `cost_model.pkl` are generated by running the notebook before starting the API.
- The API expects the same feature schema used during model training.
